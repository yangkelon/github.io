发表于2024-03-01更新于2024-07-17

前端登录

  • Cookie + Session 登录
  • Token 登录
  • SSO 单点登录
  • OAuth 第三方登录

Cookie + Session 登录

①、用户登录请求，服务器验证密码无误后，创建SessionId，并将它保存起来。

②、服务器端响应这个HTTP请求，并通过Set-Cookie头信息，将SessionId写入 Cookie 中。

③、登录完成后，后续的访问使用Cookie进行身份验证。

Token 登录

①、用户登录请求，服务器验证密码无误后，创建Token，并将它保存起来。

②、服务器端将 Token 返回给客户端。

③、登录完成后，后续的访问带上获取的Token。

SSO 单点登录

单点登录指的是在公司内部搭建一个公共的认证中心，公司下的所有产品的登录都可以在认证中心里完成，一个产品在认证中心登录后，再去访问另一个产品，可以不用再次登录，即可获取登录状态。

OAuth 第三方登录

常见的第三方登录：微信、qq、支付宝、github……
